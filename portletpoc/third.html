What is this?

This is a POC of a Portlet concept reminiscent of Java Portlets. It is a responsive, interactive concept allowing components to be reordered.

What are the textareas at the top?

The textareas at the top of each column represent the message sent back to the server to persist the order of the portlets.


As the portlets are dragged and reordered the textarea for the appropriate column is updated with the current portlet order.


This portlet ordering would be sent to the server as a personalization message when the drag operation is completed.

How do I interact with the portlets?

As implemented right now the following features are available. Note that in the final implementation not all of them may be present They seem to clash and do not really play nice with each other from a user experience standpoint.


Note these features work on iOS as well (at least in part - Hide / Show is difficult to operate on an iPhone). Try the iOS device in both portrait and landscape orientations.


Drag the portlets using the header - click (or touch) on the header of a portlet and drag it before or after another portlet in the same column to change the order in which that portlet appears in the list.


Columns of portlets - the columns represent groupings of portlets of different sizes. Some portlets may require more screen real estate than others. This feature allows for that. Notice that a portlet of one size cannot be dragged into a column with portlets of a different size.


Show / Hide - click (or touch) on the - / + icons to hide or show the content for that portlet respectively. Notice that while hidden a portlet can still be moved.


Mouse Cursor - as you hover over the portlet header, the show/hide icon or the resize handle the mouse cursor changes to indicate the action that can be taken. Note that on a touch device there really is no such thing as hover so the visual cue is not available there.


Resize - drag the handle in the lower right corner of a portlet (or right side or bottom edges) to resize it. This is not a great feature and creates lots of problems for a production worthy implementation. Note that after resize a hide / show operation currently causes the width to be maintained but not the height.


Refresh - simply refresh the page to reset to the default rendering
